JavaScript History

LiveScript to ECMAScriptv3 / 1995 - 1999

  • Brendan Eich, Netscape & Sun
  • Netscape Navigator 2
  • Self - Scheme - Java
  • Internet Explorer 3.0 - JScript
  • ECMAScript 262
- In September 1995 Netscape did release the Navigator 2.0 with LiveScript. Brendan Eich said he had to create it in 10 days, else something terrible would have happend. Through the cooperation with Sun Microsystems and due to high popularity of Java those days management decides to change the name to JavaScript. JavaScript license is by Oracle now, but the name is so common sense now, so it is used as synonimous to ECMAScript. - Languages Self and Scheme were inspirational for many concepts in JavaScript. - JavaScript was designed to be very accessible and forgiving, but also very powerful. - Durch die kurze Entwicklungszeit, schlichen sich einige "Bad Parts" ein, doch die positiven Seiten überwiegen deutlich - 1996 IE 3.0 mit JScript erscheint - 1997/98 ECMA standard 262 - ECMAScript v1/v2 - "Browser Wars" vornehmlich in DOM/HTML und CSS features. Nicht Teil der Sprache!

ECMAScript v3 / 2000 - 2009

  • IE 6 to FF 1
  • Version 4: The Non-Version
  • Mozilla is back
  • AJAX - a revival
  • v8/Chrome changes everything
  • Prototype.js, jQuery, Dojo, Mootools
  • ECMAScript 5
- Microsoft gewann die BrowserWars mit dem IE6 und beherrschte den Browsermarkt in den frühen 2000ern - Zu dieser Zeit wurde versucht die Sprache weiter zu entwickeln, dies Scheiterte jedoch an hohen ansprüchen und manglender Unterstützung seitens Vendor - ECMA 4 kam nie raus und wurde 2003 eingestellt. - 2005 passierten mehrere Dinge, die eine renaissance auslösten: - Nachdem Netscape von der Bildfläche verschwand, wurde das ECMA Gremium von Microsoft dominiert. Firefox kam raus und gewann an bedeutung so dass Mozilla (mit Brendan Eich) nun Platz in dem ECMA Gremium nahm - Die AJAX technologie kam raus und ermöglichte eine neue Art webentwicklung, wodurch JavaScript an Bedeutung gewann. - Google Chrom erschien 2006 und rief einen neuen Wettlauf aus: Performance. - Prototypejs, Dojo, Mootools und vor allem jQuery schaffen es die DOM inkompatibilitäten zu kapseln und Entwicklung zu verbessern - Nach und nach verdrängte JavaScript wieder Flash. HTML5 und iPhone beschleunigten dies. - 2009 erscheint ECMA 5: strict mode, erweiterte Object und Array methoden, JSON - Fast noch wichtiger: Konsens darüber, dass ECMA 6 kommt und besseres Schicksal haben wird als ECMA 4

Rise of JavaScript / 2009 - heute

  • ECMA TC39 / ECMA Next / ECMA Harmony
  • Backbone et al
  • HTML 5 / CSS 3
  • Node.js
  • JavaScript eats the world
  • ECMAScript 2015
- Mobile Devices killten Flash etc. - TC 39 entwickelt ECMAScript weiter, 2014/2015 fertig, viele teile bereits implementiert. Diverse zusammenstellung. - Mit stark verbesserter Perfomance, verbesserten APIs ist der Browser zu einer Rich Cient PLattform geworden - Backbonejs, Angular, Knockout etc. MVC Frameworks für den Client, Ajax und WebScockets. - HTML 5 und CSS 3 erlauben neue Möglichkeiten die Flash überflüssig machen. - Node.js brachte JavaScript auf den Server und schlug ein. - JavaScript ist überall: TV, Win8, JVM (Nashorn), Arduino, RaspberryPI, Photoshop, Datenbanken etc.etc.etc - ECMA 2015 ist ein großer sprung für die Sprache.
